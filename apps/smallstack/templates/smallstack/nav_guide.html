Navigation Guide

How to configure the sidebar, topbar, and contextual navigation areas. One registry drives everything.

Sidebar Navigation

The vertical nav panel on the left. Supports sections, submenus, and per-page control.

Three States open, closed, or disabled. Users toggle with the hamburger button.
Sections Items grouped by section: main, app, page, resources, admin.
Submenus Nested parent/child items with expand/collapse animation.
Active Detection Longest URL prefix match highlights the current page. Child active auto-opens parent.
The sidebar renders items from sections: main, app, page, resources, and admin. It skips the topbar section entirely.

Register items in your app's AppConfig.ready() method. They appear in the sidebar automatically.

apps/myapp/apps.py 
from django.apps import AppConfig

class MyAppConfig(AppConfig):
    name = "apps.myapp"

    def ready(self):
        from apps.smallstack.navigation import nav

        nav.register(
            section="main",          # appears in sidebar main section
            label="Dashboard",
            url_name="myapp:dashboard",
            icon_svg='<svg viewBox="0 0 24 24" width="20" height="20" fill="currentColor"><path d="M3 13h8V3H3v10zm0 8h8v-6H3v6zm10 0h8V11h-8v10zm0-18v6h8V3h-8z"/></svg>',
            order=10,
        )
Parameter Required Description
sectionYesmain, topbar, app, page, resources, admin
labelYesDisplay text shown in the nav
url_nameYesDjango URL name passed to reverse()
icon_svgNoInline SVG string for the icon
orderNoSort position (lower = higher). Default: 0
parentNoLabel of parent item to create a submenu
auth_requiredNoOnly show to authenticated users
staff_requiredNoOnly show to staff users

Create nested navigation by setting parent to the label of a top-level item in the same section.

apps/schedule/apps.py 
def ready(self):
    from apps.smallstack.navigation import nav

    # Parent item (renders as expandable group)
    nav.register(
        section="main",
        label="Schedule",
        url_name="schedule:index",
        icon_svg='<svg>...</svg>',
        order=10,
    )

    # Children (nested under parent)
    nav.register(
        section="main",
        label="Calendar",
        url_name="schedule:calendar",
        parent="Schedule",    # must match parent label exactly
        icon_svg='<svg>...</svg>',
        order=0,
    )
    nav.register(
        section="main",
        label="Results",
        url_name="schedule:results",
        parent="Schedule",
        icon_svg='<svg>...</svg>',
        order=1,
    )
If a child references a parent label that doesn't exist, it's promoted to a top-level item instead of being silently dropped. When a child page is active, its parent auto-opens on page load.

Sidebar state is resolved with this precedence (highest first):

  1. Template block — {% block sidebar_state %}disabled{% endblock %}
  2. View context — request._smallstack_sidebar_state = "closed"
  3. User preference — localStorage set by clicking the toggle button
  4. Global setting — SMALLSTACK_SIDEBAR_DEFAULT in settings
config/settings/base.py 
# Global default sidebar state: "open", "closed", or "disabled"
SMALLSTACK_SIDEBAR_DEFAULT = "open"
templates/mypage.html — per-page override 
{% templatetag openblock %} extends "smallstack/base.html" {% templatetag closeblock %}

{# Disable sidebar on this page (forced, ignores localStorage) #}
{% templatetag openblock %} block sidebar_state {% templatetag closeblock %}disabled{% templatetag openblock %} endblock {% templatetag closeblock %}
views.py — per-view override 
class LandingView(TemplateView):
    template_name = "landing.html"

    def get(self, request, *args, **kwargs):
        request._smallstack_sidebar_state = "disabled"
        return super().get(request, *args, **kwargs)
When state is set via template block or view context, it is forced — JavaScript won't override it with the user's localStorage preference.

Topbar Navigation

Horizontal nav in the top bar. Renders from the same registry as the sidebar.

Same Data Source Topbar reads from the nav registry. No separate config needed.
Three Modes auto (shows when sidebar closed), always, or hidden.
Section Override Register to topbar section for different items than sidebar.
Dropdowns Items with children, plus resources and admin sections, render as dropdown menus.
By default, the topbar renders main section items horizontally. If any items are registered to the topbar section, those are shown instead of main.
ModeWhen VisibleUse Case
autoWhen sidebar is closed or disabledDefault — topbar fills in when sidebar is gone
alwaysAlways, even with sidebar openDual nav surfaces for complex apps
hiddenNeverSidebar-only layout, topbar shows only logo + user
config/settings/base.py 
# Show topbar nav even when sidebar is open
SMALLSTACK_TOPBAR_NAV_ALWAYS = True   # "always" mode
SMALLSTACK_TOPBAR_NAV_ALWAYS = False  # "auto" mode (default)
templates/mypage.html — per-page override

{# Force topbar nav visible on this page regardless of global setting #}

{# Or hide it on this specific page #}

Register items to section="topbar" to show different navigation in the topbar than the sidebar. The sidebar always uses main; the topbar uses topbar items when they exist, otherwise falls back to main.

apps/website/apps.py 
def ready(self):
    from apps.smallstack.navigation import nav

    # These go in the SIDEBAR only
    nav.register(section="main", label="Dashboard",
                 url_name="dashboard", order=0)
    nav.register(section="main", label="Settings",
                 url_name="settings", order=1)

    # These go in the TOPBAR only (overrides main)
    nav.register(section="topbar", label="Features",
                 url_name="website:features", order=0)
    nav.register(section="topbar", label="Pricing",
                 url_name="website:pricing", order=1)
    nav.register(section="topbar", label="About",
                 url_name="website:about",
                 parent="More", order=0)  # dropdown child
This is useful for sites where the sidebar has admin/app navigation while the topbar shows public-facing marketing links.

In the topbar, items with children render as dropdown menus. The resources and admin sections each render as a single dropdown.

apps/website/apps.py — topbar dropdown 
def ready(self):
    from apps.smallstack.navigation import nav

    # "More" becomes a dropdown trigger in the topbar
    nav.register(section="topbar", label="About",
                 url_name="website:about",
                 parent="More", order=0)
    nav.register(section="topbar", label="Blog",
                 url_name="website:blog",
                 parent="More", order=1)
    nav.register(section="topbar", label="Contact",
                 url_name="website:contact",
                 parent="More", order=2)
Resources & Admin sections 
# Resources section renders as a "Resources" dropdown in topbar
nav.register(section="resources", label="Help",
             url_name="help:index", order=0)
nav.register(section="resources", label="API Docs",
             url_name="api:docs", order=1)

# Admin section renders as an "Admin" dropdown (staff only)
nav.register(section="admin", label="Dashboard",
             url_name="smallstack_dashboard",
             staff_required=True, order=0)
nav.register(section="admin", label="Backups",
             url_name="smallstack:backups",
             staff_required=True, order=1)

App & Page Sections

Contextual navigation areas for app-level and page-level items in the sidebar.

SectionSidebarTopbarPurpose
main Yes Yes (fallback) Primary site navigation
topbar No Yes (overrides main) Topbar-only alternate items
app Yes No App-level contextual items
page Yes No Page-level contextual items
resources Yes As dropdown Help, docs, external links
admin Yes As dropdown Staff-only administration
The app and page sections appear in the sidebar between main and resources. They can be populated via the registry (always present) or via template blocks (per-page).

Items registered to the app or page section appear in the sidebar with their own section header.

apps/reports/apps.py 
def ready(self):
    from apps.smallstack.navigation import nav

    # App-level items (always visible when app is installed)
    nav.register(
        section="app",
        label="Reports",
        url_name="reports:index",
        icon_svg='<svg>...</svg>',
        auth_required=True,
        order=0,
    )
    nav.register(
        section="app",
        label="New Report",
        url_name="reports:create",
        icon_svg='<svg>...</svg>',
        auth_required=True,
        order=1,
    )
Use section="app" for items that are always relevant to your app. Use section="page" for items that relate to a specific page or feature area.

For per-page contextual navigation, use template blocks. These render in the sidebar on that specific page only.

Template block items use raw HTML — you have full control over the markup. Use the nav-item, nav-link, and nav-section-title CSS classes to match the sidebar's built-in styling.

Full Examples

Common patterns combining sidebar, topbar, and contextual nav.

Public website with no sidebar, topbar-only navigation, and custom topbar items.

SMALLSTACK_SIDEBAR_DEFAULT = "disabled"
SMALLSTACK_TOPBAR_NAV_ALWAYS = True

def ready(self):
