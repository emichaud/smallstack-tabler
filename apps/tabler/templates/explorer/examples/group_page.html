This page has two levels: a sidebar listing all groups and a main area showing models for the selected group. Here's how to build the same pattern.

1. URLs

One route handles every group. The <str:group> capture becomes self.kwargs["group"] in the view.

apps/myapp/urls.py 
from django.urls import path
from .views import GroupDashboardView

urlpatterns = [
    path("dashboard/<str:group>/", GroupDashboardView.as_view(), name="myapp-group-dashboard"),
]

2. View

ExplorerGroupMixin populates group_name, models, and all_groups.

apps/myapp/views.py 
from django.views.generic import TemplateView
from apps.smallstack.mixins import StaffRequiredMixin
from apps.explorer.mixins import ExplorerGroupMixin

class GroupDashboardView(ExplorerGroupMixin, StaffRequiredMixin, TemplateView):
    template_name = "myapp/group_dashboard.html"

3. Register models with a group

Set explorer_group on any ModelAdmin to assign it to a group.

apps/heartbeat/admin.py 
@admin.register(Heartbeat)
class HeartbeatAdmin(admin.ModelAdmin):
    list_display = ("checked_at", "status", "response_time_ms")
    explorer_enabled = True
    explorer_group = "Monitoring"
