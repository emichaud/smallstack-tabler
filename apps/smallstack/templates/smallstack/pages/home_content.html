{# ── ONE MODEL → THREE SURFACES (the killer feature) ────────────── #}
The pipeline

One model. Three surfaces.

Define your model once. CRUDView's opt-in flags emit the same form/queryset logic as an HTML admin page, a REST endpoint, and an MCP tool — wired through the same tenancy hook and permission checks. 

# apps/support/views.py
class TicketCRUDView(CRUDView):
    model           = Ticket
    actions         = [Action.LIST, Action.CREATE, Action.DETAIL, Action.UPDATE, Action.DELETE]
    filter_fields   = ["status", "priority", "customer"]
    enable_api      = True     ───→  REST  /api/tickets/
    enable_mcp      = True     ───→  MCP   list_tickets, get_ticket, create_ticket, …
    enable_explorer = True     ───→  HTML  /smallstack/explorer/support/ticket/
HTML

Explorer

Browsable list / detail / create / update / delete, themed to your project.

REST

/api/<resource>/

JSON with Bearer tokens, search, filter, ordering, FK expansion, aggregation, exports.

MCP

/mcp · tools/list

JSON-RPC + OAuth. Claude Desktop and Connectors UI work without extra setup.
