This page groups models by their Django app_label instead of a custom explorer_group. The sidebar lists all apps that have Explorer-registered models. Clicking an app shows its models. Here's how to build the same pattern.

1. URL

One route handles every app. The <str:app_label> capture becomes self.kwargs["app_label"] in the view, which the mixin passes to the registry to look up models for that Django app.

apps/myapp/urls.py 
from django.urls import path
from .views import AppDashboardView

# Pattern:
#   /dashboard/app/<app_label>/
#
# <str:app_label> captures the Django app label from the URL.
# The mixin reads it via self.kwargs["app_label"] and passes
# it to explorer_registry.get_app_context(app_label),
# which returns all models belonging to that app.
#
# The app_label comes from the model's Meta class:
#   class Meta:
#       app_label = "heartbeat"  →  /dashboard/app/heartbeat/
#
# Example: visiting /dashboard/app/heartbeat/ returns:
#   - app_label:        "heartbeat"
#   - app_verbose_name: "Heartbeat"
#   - models:           [Heartbeat, HeartbeatEpoch, HeartbeatDaily, ...]
#   - all_apps:         ["accounts", "heartbeat", ...]

urlpatterns = [
    path(
        "dashboard/app/<str:app_label>/",
        AppDashboardView.as_view(),
        name="myapp-app-dashboard",
    ),
]

2. View — with mixin (recommended)

ExplorerAppMixin handles everything: reads the app_label from the URL, calls the registry, 404s if not found, and populates app_label, app_verbose_name, models, and all_apps into the template context. Your view is just two lines.

apps/myapp/views.py 
from django.views.generic import TemplateView
from apps.smallstack.mixins import StaffRequiredMixin
from apps.explorer.mixins import ExplorerAppMixin


# Dynamic — reads app_label from URL parameter
class AppDashboardView(ExplorerAppMixin, StaffRequiredMixin, TemplateView):
    template_name = "myapp/app_dashboard.html"


# Hardcoded — always shows "heartbeat" app, no URL param needed
class HeartbeatAppView(ExplorerAppMixin, StaffRequiredMixin, TemplateView):
    template_name = "myapp/heartbeat_app.html"
    explorer_app = "heartbeat"

2b. View — manual (if you need custom logic)

Use get_app_context() directly if you need to add extra data or transform the results. Call .as_context() to get a dict ready for context.update().

apps/myapp/views.py 
from django.http import Http404
from django.views.generic import TemplateView
from apps.smallstack.mixins import StaffRequiredMixin
from apps.explorer.registry import explorer_registry


class AppDashboardView(StaffRequiredMixin, TemplateView):
    template_name = "myapp/app_dashboard.html"

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        ctx = explorer_registry.get_app_context(self.kwargs["app_label"])
        if not ctx:
            raise Http404("App not found.")
        context.update(ctx.as_context())
        # Add your own extra data here
        context["my_custom_thing"] = "whatever you need"
        return context

4. Groups vs Apps — when to use each

Explorer offers two ways to organize models into sidebar pages. Choose the one that fits your use case.

Comparison 
# GROUPS (ExplorerGroupMixin)
# - Custom labels you define via explorer_group on ModelAdmin
# - Models from different apps can share the same group
# - URL: <str:group>         e.g. /dashboard/Monitoring/
# - Mixin: ExplorerGroupMixin
# - Registry: get_group_context(group_name)
# - Best for: cross-app dashboards, custom categories
#
# Example: Heartbeat + RequestLog both in "Monitoring" group
#   class HeartbeatAdmin:
#       explorer_group = "Monitoring"
#   class RequestLogAdmin:
#       explorer_group = "Monitoring"


# APPS (ExplorerAppMixin)
# - Follows Django's built-in app_label structure
# - Models grouped by the app they belong to
# - URL: <str:app_label>     e.g. /dashboard/app/heartbeat/
# - Mixin: ExplorerAppMixin
# - Registry: get_app_context(app_label)
# - Best for: admin-style navigation, per-app pages
#
# No extra config needed — app_label comes from the model:
#   class Heartbeat(models.Model):
#       class Meta:
#           app_label = "heartbeat"  # automatic from app directory
