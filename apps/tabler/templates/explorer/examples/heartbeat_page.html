This page demonstrates the real power of composability — mixing Explorer's auto-generated CRUD table with your own custom visuals on the same page. The mixin provides the data table; you add whatever else the page needs.

1. The idea

ExplorerModelMixin gives you a full CRUD table for any registered model. But your page isn't just a table — you want charts, stats, alerts, or any other visual alongside it. The mixin handles the boring CRUD plumbing so you can focus on the interesting parts.

2. View — mixin + custom data

Hardcode the model with explorer_app_label and explorer_model_name, then override get_context_data to add your chart data alongside the CRUD context the mixin already provides.

apps/myapp/views.py 
from django.views.generic import TemplateView
from apps.smallstack.mixins import StaffRequiredMixin
from apps.explorer.mixins import ExplorerModelMixin


class HeartbeatPageView(ExplorerModelMixin, StaffRequiredMixin, TemplateView):
    template_name = "myapp/heartbeat_page.html"

    # Hardcoded model — no URL params needed
    explorer_app_label = "heartbeat"
    explorer_model_name = "heartbeat"

    def get_context_data(self, **kwargs):
        context = super().get_context_data(**kwargs)
        from apps.heartbeat.models import HeartbeatDaily
        context["chart_days"] = HeartbeatDaily.get_daily_summary(days=7)
        return context

3. Template

The template has two sections: your custom chart at the top, and the standard crud_table tag below.

4. URL

Since the model is hardcoded in the view, the URL doesn't need any parameters.

apps/myapp/urls.py 
from django.urls import path
from .views import HeartbeatPageView

urlpatterns = [
    path("heartbeats/", HeartbeatPageView.as_view(), name="heartbeat-page"),
]
